# Product

## Product Name
Developer Marketer Portfolio

## Product Purpose
A dual-persona single-page portfolio for a Marketing Technology Consultant applying to PostHog's Developer Marketer role. The core mechanic is a MARKETEER / DEVELOPER mode toggle that demonstrates both sides of the candidate's competency in a single artifact.

## Register
brand — the design IS the product. The site is an application itself, not a wrapper for a tool.

## Target Audience
PostHog hiring team (primary). The site is addressed directly to the reader — "I Make Developers Give a Damn" — with PostHog as the named audience in a kinetic word-swap cycle.

## Users
One specific hiring team. The portfolio is a creative brief, not a general portfolio. It is designed to be memorable, not comprehensive.

## Brand Tone
- Developer mode: precise, technical, dry wit. GitHub dark aesthetic. Code-as-identity.
- Marketeer mode: bold, editorial, self-aware. Brutalist typography. The energy of a great campaign brief.
- Shared: confident without being arrogant. Human without being casual. The em dash is banned — use commas, colons, or parentheses.

## Anti-references
- Generic SaaS portfolio sites (template-look, hero-metric template, gradient text)
- Glassmorphism decorative cards
- Blue-gradient dark mode "developer" aesthetic
- LinkedIn-style profile pages

## Strategic Principles
1. The toggle IS the brief. One person, two modes, one point proven.
2. Every section in Developer mode looks like a codebase; every section in Marketeer mode looks like a campaign.
3. PostHog orange (#F54E00) is a thread through the Developer mode — the product lives in the design.
4. The Marketeer mode ends with "// also available in developer flavour" — a wink to the reader.
